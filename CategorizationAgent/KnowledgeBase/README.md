# 고객 문의 분류 및 대응 가이드

## 0. 공통 대응 스키마

모든 카테고리는 아래 구조를 따릅니다:

```jsonc
{
  "category_id": 1,
  "name_ko": "카테고리명",
  "name_en": "CategoryName",
  
  // 상담/툴에서 내부적으로 보는 체크리스트
  "handling": {
    "when_to_use": "이 카테고리가 선택되는 조건 설명",
    "data_to_collect": ["필요 기본 정보 목록"],
    "checks": [
      "조회/검증해야 하는 항목 순서대로",
      "DB, 로그, 상태 확인 등"
    ],
    "decision_rules": {
      "case_1": "상황별 처리 기준 요약",
      "case_2": "..."
    }
  },
  
  // 실제 고객에게 보내는 답변 양식(템플릿)
  "reply_template_ko": {
    "normal": "정상 케이스 답변",
    "not_found": "데이터가 없는 경우 답변",
    "error": "오류 발생 시 답변"
  }
}
```

## 사용 방법

1. **분류 모델**이 문의 내용을 분석하여 `category_id`를 반환
2. 해당 `category_id`로 위 가이드에서 대응 블록을 찾음
3. `handling` 섹션의 체크리스트에 따라 필요한 데이터 조회 및 검증 수행
4. 상황(case)에 맞는 `reply_template_ko`를 선택하여 고객에게 답변 전송

## 참고 사항

- 모든 답변 템플릿은 고객의 상황에 맞게 변수({account}, {date}, {compensation} 등)를 치환하여 사용
- 예외 상황 발생 시 개발팀 또는 정책팀과 협의하여 처리

- 정책에 따라 보상 지급 여부 및 금액은 별도로 결정
